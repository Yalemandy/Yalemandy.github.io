前言

顺序表我们进入数据结构的开头，对后面的数据结构有一定的关系，顺序表是属于线性表中的一种还有别的数据结构，如：栈、队列、树……等等

一、线性表的特性

①物理特性：不一定连续

②逻辑特性：连续

二、顺序表的特性

①物理特性：连续

②逻辑特性：连续

三、顺序表的分类

①静态顺序表：

struct SeqList

{

​ int arr[100]; //定长数组

​ int size; //顺序表当前有效的数据个数

};

②动态顺序表：

struct SeqList

{

​ int* arr; //定义一个指针，让他指向这个顺序表的开头

​ int size; //顺序表当前有效的数据个数

​ int capacity;//当前顺序表的空间大小

};

在定义之前我们需要想一下，如果一开始定死了就是int类型，那就太局限了

所以我们在声明顺序表的时候就可以加上这么一句：

typedef int SLTDataType;

在需要换的时候直接在这里一次更换，全局都会改变，就会大大的提升了效率

四、顺序表的选择

如果用静态顺序表来完成顺序表，就会太过局限，比如是开始就给一个特别的大的空间，这样的话，那后期如果用不上这么多空间岂不是浪费了太多的空间。

所以选择使用动态顺序表来完成顺序表。

五、顺序表的实现

在实现顺序表之前，我们需要创建三个文件，两个.c文件和一个.h文件：SeqList.c（代码的实现）、text.c（代码的测试）、SeqList.h（所有函数的定义以及库函数的引入）。
